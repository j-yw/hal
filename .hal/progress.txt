## Codebase Patterns
- DigitalOceanProvider follows the same `cmdContext` injection pattern as Hetzner/Daytona — override in tests to capture args without running doctl.
- The `ensureDoctl()` helper checks `exec.LookPath("doctl")` and is called at the top of every DigitalOceanProvider method to fail fast with install instructions.
- Cloud-init generation for DigitalOcean (`generateDOCloudInit`) uses the same format as Hetzner — env vars in `/etc/environment`, basic dev packages.
- `buildDOCreateArgs` returns the args slice for `doctl compute droplet create` — extracted as a pure function for testability.
- Provider implementations (DaytonaProvider, HetznerProvider) use an injectable `cmdContext` function field for testability — tests swap it with a closure that records args and returns a mock command.
- `resolveProviderFromState` in `cmd/sandbox.go` is the shared factory used by stop/delete/status/ssh to build a Provider from sandbox state + config.
- Mock providers in cmd tests implement the full `sandbox.Provider` interface with only the relevant method populated — keep each mock focused on one method.
- Cloud-init generation (`generateCloudInit`) is a pure function — easy to unit test independently from provider Create.
- The `stripDashDash` helper in `cmd/sandbox_ssh.go` normalizes args for the ssh command — `DisableFlagParsing: true` on the Cobra command passes all args through.
- `lastSSHCmd` package-level var in `cmd/sandbox_ssh.go` captures the cmd in test mode — avoids real `syscall.Exec` in tests.
- Use `cmd.Root()` as the public accessor for the Cobra command tree when tooling needs to reuse runtime command wiring.
- Keep CLI docs generation in a dedicated `internal/tools/docgen` binary and always set `root.DisableAutoGenTag = true` before calling Cobra doc generators.
- Implement `make docs-cli` as a temp-output generation + directory replace flow (`docs/cli.tmp` -> `docs/cli`) to eliminate stale markdown pages deterministically.
- Implement `make docs-check` by regenerating markdown docs into a clean temp directory and running a recursive diff against `docs/cli` so both content drift and stale files fail fast.
- In CI, run `make docs-check` with `make vet` and `make test` so docs drift and metadata contract regressions fail on pull requests.
- Verify docs-baseline determinism by running `make docs-cli` twice and ensuring no `docs/cli` diff before marking docs artifact stories complete.
- Keep command metadata-contract scope in a shared helper that excludes hidden/deprecated and Cobra help-only commands (`help [command]` and `IsAdditionalHelpTopicCommand()`), while still including non-runnable parent commands that own in-scope child pages.
- Keep user-facing Cobra command examples in `Example` fields and verify required metadata (`Use`, `Short`, `Long`, `Example`) with table-driven tests rooted at `cmd.Root()`.
- For command-family metadata checks, recurse over in-scope descendants under each top-level family and assert each `Example` includes that command's path.
- Add a global metadata contract test that recursively walks from `cmd.Root()` and reports missing `Use`/`Short`/`Long`/`Example` fields per command path.
- Keep a dedicated README `CLI Reference` section that links to `docs/cli/` and `docs/cli/hal.md` so generated command docs remain discoverable.
