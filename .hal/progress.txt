## Codebase Patterns

- Factory durable schema additions should preserve compatibility fields like `RunRecord.SandboxName` while adding richer optional objects; lock JSON field names and omission behavior in `internal/factory/types_test.go`.
- Factory machine contract docs are guarded by `cmd/contracts_doc_test.go`; add reflection-backed JSON tag checks there when documenting nested durable metadata objects so docs drift fails tests.
- Sandbox-backed factory execution side effects belong behind `factorySandboxExecutorDeps` in `cmd/factory_sandbox_executor.go`; add new remote behavior through normalized injectable dependencies so tests can fake registry, provisioning/start, provider exec, clocks, and store persistence.
- `hal factory run --sandbox` selection lives in `factoryRunRequest.Sandbox` and dispatches through `factoryRunDeps.runSandbox`; keep local execution behind `runPipeline` so no-flag factory runs stay on the existing local path.
- Existing sandbox reuse in `runFactorySandboxExecutorWithDeps` should persist `RunRecord.SandboxName` and `RunRecord.Sandbox` immediately after target resolution/start; no-name resolution should keep `sandbox.ResolveDefault` running-only error strings unless a later provisioning path intentionally handles them.
- Sandbox provision/start failures should be persisted inside `runFactorySandboxExecutorWithDeps` with `RunStatusFailed`, `FailureCategoryPipeline`, and sandbox handoff metadata before returning the wrapped error to the outer factory runner.
