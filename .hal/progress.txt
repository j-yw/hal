## Codebase Patterns
- Postgres/Turso ClaimRun uses `FOR UPDATE SKIP LOCKED` subquery pattern for atomic claim ‚Äî modifications go in the outer UPDATE SET clause.
- Integration tests behind `//go:build integration` tags need `go vet -tags=integration` to verify compilation since default `go test ./...` skips them.
- `TransitionRun(claimed -> queued)` is valid for re-queuing runs in tests to simulate retry scenarios.
- `scanRun` reads all RETURNING columns including `attempt_count`, so adding SET clauses to the UPDATE is automatically reflected in the returned Run struct.
- Turso store tests are regular unit tests (no build tags) using `modernc.org/sqlite`, while Postgres store tests require `//go:build integration` and `HAL_TEST_POSTGRES_DSN`.
- After US-001/US-002 made store adapters atomically increment `attempt_count`, ClaimService must use `run.AttemptCount` directly (not `+1`) for `Attempt.AttemptNumber` since the store already incremented it.
- Mock `claimedRun()` in claim_test.go represents the run as returned by `ClaimRun` (post-increment), so `AttemptCount` should reflect the incremented value (e.g., 1 for first claim, not 0).
- Use `cloud.ShellQuote(s)` from `internal/cloud/shell.go` for safe single-quote shell escaping ‚Äî wraps in `'...'` with `'` ‚Üí `'\''` replacement. The old `escapeShellSingleQuote` in auth_materialization.go only escapes inner content without wrapping.
- `CollectSandboxBundle` uses `MatchesArtifactPatterns` (workflow-aware filtering) not `IsBundlePathAllowed` (input allowlist) ‚Äî these are two different filtering policies for worker-side vs client-side collection.
- When building shell commands with `ShellQuote`, wrap the entire path argument (e.g., `ShellQuote("/workspace/" + relPath)`) not just part of it.
- Use `mockCollectRunner` + `sandboxFS` pattern for testing sandbox exec operations without real infrastructure.
- `readSandboxFile` uses `stripBase64Whitespace` to strip all whitespace from base64 output before decoding ‚Äî handles wrapped output from non-GNU base64 implementations and trailing newlines.
- `base64Cmd` is a package-level variable (`"base64 -w0"`) used by `readSandboxFile` ‚Äî mock runners must use `base64Cmd` (not hardcoded `"base64 -w0"`) when matching command prefixes.
- Bundle framing uses `path\x00size\x00content` (null-terminated path, null-terminated decimal size, raw content bytes) plus gzip ‚Äî `CompressBundle` (worker-side) and `compressBundleFiles` (client-side) produce identical framing consumed by `decompressBundleFiles`.
- Use `ComputeSandboxBundleHash(records)` for worker-side snapshot SHA ‚Äî it converts `SandboxBundleRecord` ‚Üí `BundleManifestRecord` via `NewBundleManifestRecord` then delegates to `ComputeBundleHash`. Never hash compressed payload bytes for the snapshot SHA field.
- `GitHubPRCreator(r, sandboxID, authDir)` returns a `PRCreator` closure ‚Äî use `mockPRRunner` in tests to capture and assert the executed command text without real sandbox infrastructure.
- `parseGHPROutput` extracts the PR URL from `gh pr create` stdout by scanning the last non-empty line ‚Äî `gh` may emit progress text before the URL.
- `WorkerPipeline` uses config-struct constructor pattern (`WorkerPipelineConfig` ‚Üí `NewWorkerPipeline`) with explicit nil/empty checks for required dependencies (Store, Runner, WorkerID, Claim).
- `ProcessOne` maps `ErrNotFound` from `ClaimAndLock` to `ErrNoWork` ‚Äî callers should use `IsNoWork(err)` to detect empty queue, not `IsNotFound`.
- Worker tests use `workerMockStore` embedding `mockStore` plus `workerMockRunner` implementing `runner.Runner` ‚Äî reuse this pattern for future worker-related tests.
