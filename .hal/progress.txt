## Codebase Patterns
- Config env var constants live in `internal/cloud/deploy/config.go` â€” add new env constants to the same const block.
- `LoadConfig(getenv)` pattern enables testable config loading; tests use inline `func(key string) string { return env[key] }` closures.
- `ValidateStore()` is DB-only validation; `Validate()` is full validation. Both delegate to `validateAdapter()` for DB checks.
- URL fallback precedence is implemented directly in `LoadConfig` (check primary, fallback to legacy alias).
- Config fields are value types (not pointers) since these come from env vars, not YAML.
- SDK client constructor pattern: `SDKClientConfig` struct â†’ pre-validate required fields â†’ delegate to `daytona.NewClientWithConfig` â†’ wrap SDK errors with `%w`.
- SDK client tests are in `sdk_client_test.go` (package-internal); HTTP client tests remain in `runner_test.go`.
- The `daytona.NewClientWithConfig` constructor does not make network calls â€” it only configures HTTP clients, so tests can construct real SDK clients without mocking.
- SDK sandbox lifecycle: `client.Create` â†’ `*daytona.Sandbox`, `client.Get(id)` â†’ `*daytona.Sandbox`, then `sandbox.Delete(ctx)` â€” the delete is on the sandbox object, not the client.
- Use cancelled-context tests (`ctx, cancel := ...; cancel()`) to exercise SDK error wrapping without a mock or live server.
- SDK `sandbox.Process.ExecuteCommand(ctx, cmd, opts...)` returns `*types.ExecuteResponse{ExitCode, Result, Artifacts}` â€” map `Result` to stdout, prefer `Artifacts.Stdout` when available.
- SDK options are functional: `options.WithCwd(dir)`, `options.WithExecuteTimeout(dur)` â€” build a `[]func(*options.ExecuteCommand)` slice and spread with `opts...`.
- SDK `ProcessService.GetSessionCommandLogs(ctx, sessionID, commandID)` returns `map[string]any` â€” access logs via `logsMap["logs"].(string)` with zero-value fallback.
- SDK `client.List(ctx, labels, page *int, limit *int)` returns `*PaginatedSandboxes` â€” use for health probes with page=1, limit=1 and `daytona.Version` for version info.
