# Ralph Progress Log

## Codebase Patterns

- Factory bootstrap request/result DTOs live in `internal/factory`; add explicit JSON tags and lock exported machine-readable fields with raw-key plus round-trip tests in `internal/factory/types_test.go`.
- Bootstrap failure classification lives in `internal/factory/bootstrap_failure.go` with a separate bootstrap taxonomy (`repo`, `auth`, `dependency`, `engine_setup`, `unknown`) and sanitized messages that avoid raw command output.
- Factory bootstrap command execution goes through `internal/factory.BootstrapCommandExecutor` and `RunBootstrapStep`; tests should inject fake executors and deterministic clocks rather than spawning git, Hal, or engine CLIs directly.
- Tooling verification goes through `internal/factory.BootstrapVerifyTooling`; configure required Hal/engine checks with `BootstrapToolingCheck`, and only attach installer commands when `BootstrapOptions.InstallMissingCLIs` should allow install-and-recheck behavior.
- Repository checkout bootstrap goes through `internal/factory.BootstrapRepositoryCheckout`; inject `RepoExists`, executor, and clock dependencies so clone/fetch/checkout command construction is tested deterministically without touching real repositories.
- Run branch preparation stays in `BootstrapRepositoryCheckout` after base checkout; inject `LocalBranchExists`/`RemoteBranchExists` probes so tests cover local retry, remote resume, and first-run branch creation without real git refs.
- Hal template, standards, managed skill, and engine-link setup goes through `internal/factory.BootstrapRefreshHal`; delegate to `hal init` or `hal init --refresh-templates` plus `hal links refresh` through the executor boundary instead of duplicating template or skill ownership logic.
- Bootstrap request environment values flow through `BootstrapStepDeps.Request`; use `RunBootstrapStep` so request env reaches the executor while returned command summaries and command results are sanitized.
- Use `NewBootstrapSanitizer` or the `SanitizeBootstrapCommand*` helpers before recording bootstrap command/timeline data; sensitive env keys and their configured values must not appear in serialized records.
