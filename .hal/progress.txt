# Ralph Progress Log

## Codebase Patterns

- Factory queue JSON contracts are split between reusable durable types in `internal/factory` (`QueueEntry`, `QueueClaim`, `QueueStatus*`) and command response DTOs in `cmd/factory_queue_contracts.go`; keep docs/examples/tests aligned when implementing queue commands.
- Factory queue storage builds on `internal/factory.Store`: keep durable queue state at the global config-backed factory root (`StoreDir()/queue.json`), return empty state for missing files without creating directories, and preserve corrupt files by returning parse errors without mutation.
- Queue read-modify-write paths should use `Store.UpdateQueue`; it holds `queue.lock` across load, mutation, and temp-file-plus-rename save so concurrent local queue workers preserve all updates.
- Queue commands should use `Store.EnqueueQueueEntry`, `Store.ListQueue`, and `Store.ClaimNextQueueEntry` for FIFO behavior; tests can inject `QueueOperationOptions` for deterministic timestamps and queue IDs.
- Queue lifecycle transitions should go through Store helpers (`ClaimNextQueueEntry`, `MarkQueueEntrySucceeded`, `MarkQueueEntryFailed`) so claim metadata, attempt counts, terminal timestamps, and retained queue history stay consistent.
- Queue command implementations should validate executor modes through `factory.ValidateExecutorMode`, enqueue through Store helpers, and record queue-related run/timeline state in `cmd` so queue state and factory run history stay synchronized.
- Queue work claiming should use `Store.ClaimNextQueueEntry`, then record the associated run `CurrentStep` as `claimed` plus a claim timeline event; leave actual executor execution to the worker execution path.
- Factory queue command registration lives in `cmd/factory_queue.go`; wire the command group from `cmd/factory.go`, update `cmd/factory_test.go` metadata/link checks, and run `make docs-cli` when changing CLI docs.
