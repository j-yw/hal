# Ralph Progress Log

## Codebase Patterns

- Use `template.ProductDir`, `template.ProductMissionFile`, `template.ProductRoadmapFile`, and `template.ProductTechStackFile` for product-doc paths instead of hardcoded `.hal/product/*` strings.
- Use `template.ProductFiles()` when iterating product docs so mission/roadmap/tech-stack order stays deterministic.
- In `internal/product.GeneratedPayload`, keep file content fields as `*string` keyed by filename JSON tags (`mission.md`, `roadmap.md`, `tech-stack.md`) so omitted unselected files stay distinguishable from intentionally empty strings.
- Use `product.LoadExistingFiles(projectDir)` for preflight reads; missing product files should map to `Exists=false` + empty content without creating `.hal/product`.
- Use `product.WriteSelectedFiles(projectDir, targets, payload)` for selective product updates; it only writes selected files with generated content and creates `.hal/product` lazily when writes are needed.
- For Hal-managed skills, keep `internal/skills/embed.go` `//go:embed` entries and `ManagedSkillNames` in sync with on-disk `internal/skills/<name>/SKILL.md`, and lock with a `LoadSkill("<name>")` non-empty test.
- For new command families, keep Cobra handlers thin and route logic through `run<Command>WithDeps` plus an options struct so flag wiring can be unit-tested without full command execution.
- In `runProductPlanFlowWithDeps`, perform `.hal` existence checks before reading product docs, and only prompt for replace/update/cancel when at least one product file already exists.
- In multi-step interactive flows, wrap `opts.In` once in a shared `*bufio.Reader` before calling multiple prompt helpers so scripted input is consumed consistently across prompts.
- Keep update-target parsing centralized in `parseProductPlanTargets`; accept concise tokens (`m/r/t`, combined letters, comma/space-delimited values) and return deterministic validation errors.
- Keep product interview collection centralized in `collectProductPlanAnswers`; ask only selected target sections and fall back to deterministic per-target default answers when input is empty/EOF.
- Keep generation prompt assembly centralized in `buildProductPlanGeneratePrompt`; include only selected targets' interview/existing context and enforce JSON-only filename-keyed output contracts.
- Keep product payload generation centralized in `generateProductPlanPayloadWithDeps`: parse strict JSON with `ParseGeneratedPayload`, run exactly one repair prompt on parse failure, and return actionable errors instead of writing partial output.
- In `runProductPlanFlowWithDeps`, derive changed-file summaries from selected targets plus non-nil payload fields before writing, then print only those created/updated files so CLI output matches actual writes.
- Keep `ensureGitignore` as the single migration point for Hal committable exceptions and update `.hal/*` plus every exception (`!.hal/standards/`, `!.hal/commands/`, `!.hal/product/`) together to preserve idempotent `hal init` behavior.
- In `internal/archive`, treat `template.ProductDir` as a protected directory: create/has-state checks must exclude product docs and restore must skip archived `product/` entries so `.hal/product/*` remains durable context.
