# Ralph Progress Log

## Codebase Patterns

- Use `template.ProductDir`, `template.ProductMissionFile`, `template.ProductRoadmapFile`, and `template.ProductTechStackFile` for product-doc paths instead of hardcoded `.hal/product/*` strings.
- Use `template.ProductFiles()` when iterating product docs so mission/roadmap/tech-stack order stays deterministic.
- In `internal/product.GeneratedPayload`, keep file content fields as `*string` keyed by filename JSON tags (`mission.md`, `roadmap.md`, `tech-stack.md`) so omitted unselected files stay distinguishable from intentionally empty strings.
- For Hal-managed skills, keep `internal/skills/embed.go` `//go:embed` entries and `ManagedSkillNames` in sync with on-disk `internal/skills/<name>/SKILL.md`, and lock with a `LoadSkill("<name>")` non-empty test.
