## Codebase Patterns
- Cloud config lives in internal/cloud/config; use config.Parse(data) or config.Load(path) for YAML cloud.yaml handling
- Use template.CloudConfigFile constant for the cloud.yaml filename
- ValidationErrors type implements error interface and collects multiple field-level failures with path/rule/remediation
- Secret detection walks raw YAML map recursively; checks both key names AND string values for secret-bearing URL query parameters
- Profile.Wait uses *bool (pointer) to distinguish omitted from explicit false — follows the pointer-yaml-defaults pattern from AGENTS.md
- ArtifactGroup (state/reports/all) follows the same typed string enum pattern as WorkflowKind and RunStatus — use NewArtifactMetadata(kind) to build per-workflow metadata
- BundleAllowlist is for input uploads; artifact groups are for output pull — they overlap but are distinct concerns (auto-state.json is output-only)
- WorkflowArtifactGroups maps workflow kinds to their default artifact groups: run→[state], auto/review→[state,reports]
