# Ralph Progress Log

## Codebase Patterns

- Factory machine-readable contracts should be locked in code, docs, and examples together: define the command response struct/renderer in `cmd`, document fields in `docs/contracts/<contract>.md`, add an example in `docs/contracts/examples/`, and cover it in `cmd/factory_test.go`, `cmd/machine_contracts_test.go`, and `cmd/contracts_doc_test.go`.
- Factory command parser changes should update Cobra metadata, focused `cmd/factory_test.go` registration/validation tests, and regenerated `docs/cli` pages via `make docs-cli`; docs drift is enforced by `make docs-check`.
- Factory run execution should stay dependency-injected at the command layer: create a pending `factory.RunRecord`, persist the running transition, then invoke the pipeline dependency so tests can use `factory.NewStore(t.TempDir())` without real engines or CLIs.
- Factory-to-auto execution should test the handoff boundary, not the real pipeline: map markdown PRDs to auto positional args, report sources to auto `--report`, and propagate `--base` through an injected auto runner.
- Factory run timeline events are recorded in the command-layer wrapper around `factoryRunDeps.runPipeline`; use the injected `RecordProgress` hook for progress events and keep completion/failure lifecycle events outside the wrapped `hal auto` implementation.
- Factory run artifact references are collected in the `cmd/factory.go` wrapper after `factoryRunDeps.runPipeline` returns; explicit source paths and the factory store run-record path are always relevant, while optional `.hal` artifacts should be detected with template constants and skipped when absent.
- Factory run final results should be rendered from the persisted `factory.RunRecord` plus `Store.LoadEvents` after terminal status/final timestamps are saved, so command output matches durable state.
