## Codebase Patterns
- Global sandbox state paths resolve in strict order: `HAL_CONFIG_HOME` â†’ `XDG_CONFIG_HOME/hal` â†’ `HOME/.config/hal`.
- For sandbox global-path tests, use `t.Setenv("HAL_CONFIG_HOME", tmpDir)` for isolation; set `HOME` explicitly when validating fallback behavior.
- Global directory setup helpers should use `os.MkdirAll(..., 0700)` and stay idempotent.
- For UUIDv7 generation in `internal/sandbox`, keep the source injectable (`clock` + `io.Reader`) and preserve monotonic ordering by incrementing random bits when timestamps do not advance.
- Keep sandbox naming canonical through `ValidateName` with exact errors (`must be 1-59 chars`, `must be lowercase alphanumeric and hyphens`, `must not start or end with hyphen`, `must not contain consecutive hyphens`) so CLI and tests share one contract.
- `BatchNames(base, count)` should enforce `count >= 1`, use zero-padding width `max(2, digits(count))`, preflight base+suffixed length against the 59-char limit, and validate each generated name via `ValidateName`.
- Sandbox lifecycle strings should use shared constants from `internal/sandbox/types.go` (`StatusRunning`, `StatusStopped`, `StatusUnknown`) to avoid status-value drift across commands and providers.
- Global registry persistence in `internal/sandbox/registry.go` should store one `sandboxes/{name}.json` file per instance with atomic temp-file + rename writes at `0600`, and keep the collision error exact: `sandbox "<name>" already exists`.
- Global sandbox config should be modeled in `internal/sandbox/globalconfig.go` with pointer-based raw YAML structs merged into `DefaultGlobalConfig()` so missing keys keep defaults while explicit zero/empty values are preserved.
- `SaveGlobalConfig()` should write `sandbox-config.yaml` atomically (`.tmp` + `os.Rename`) with `0600` permissions.
- `hal sandbox setup` should load defaults from `sandbox.LoadGlobalConfig()` and persist via `sandbox.SaveGlobalConfig()` so setup works even when no `.hal/` directory exists.
- Sandbox-setup command tests should isolate global writes with `t.Setenv("HAL_CONFIG_HOME", tmpDir)` (and clear XDG/HOME fallbacks) to avoid leaking into user-level config during test runs.
- `ListInstances()` should treat a missing global `sandboxes/` directory as an empty result and always return instances sorted by `Name`.
- `ResolveDefault(filter)` should centralize no-arg sandbox selection with deterministic contracts: exact empty-state errors, sorted-name ambiguity errors, and a returned hint string for single-match auto-connect flows.
- Sandbox config migration should treat global `sandbox-config.yaml` as authoritative: only copy local `.hal/config.yaml` sandbox/daytona sections when global is missing, and never mutate the local file.
- Commands that should continue even if migration fails should call `runSandboxAutoMigrate(projectDir, out)` so migration errors are downgraded to a warning line (`warning: sandbox migration failed: <error>`).
- Provider lifecycle/connection methods (`Stop`, `Delete`, `Status`, `SSH`, `Exec`) accept `*sandbox.ConnectInfo`; command flows should build it with `sandbox.ConnectInfoFromState(instance)` and set explicit fallback IDs when targeting raw workspace identifiers.
- During ConnectInfo migration, providers should consume `info.Name`/`info.IP` directly and avoid loading `.hal/sandbox.json` as a fallback.
- After removing the last provider-level local-state fallback, delete obsolete shared config plumbing (for example `ProviderConfig.StateDir`) and related command wiring to keep provider contracts minimal.
- Start command persists new sandboxes to the global registry (`SaveInstance`) with UUIDv7 ID, Status="running", TailscaleHostname, and CreatedAt; backward compat local `.hal/sandbox.json` write is best-effort.
- Use `mergeGlobalSizeDefaults(localCfg, globalCfg)` to fill empty provider-specific size fields from global config before building ProviderConfig in the start command.
- Start command tests should set `HAL_CONFIG_HOME` via `t.Setenv` (either in `setupStartTest` or per-test) to isolate global registry writes from user-level config.
- Auto-shutdown env injection (`HAL_AUTO_SHUTDOWN`, `HAL_IDLE_HOURS`) is always applied to merged env before `provider.Create`; use `resolveAutoShutdown(globalCfg, opts)` and `injectAutoShutdownEnv(env, auto, hours)` as the canonical path.
- Flag overrides for auto-shutdown use pointer-based `autoShutdownOpts` to distinguish "flag not set" from explicit values; `--no-auto-shutdown` always takes precedence over `--auto-shutdown`.
- Batch creation uses `batchPreflight(base, count)` to generate names via `BatchNames` and validate against the global registry before any `provider.Create` call. Preflight failures block all creation.
- `runSingleCreate` is the extracted helper for single sandbox creation; `runBatchCreate` iterates over preflight-validated targets and delegates to `runSingleCreate` for each.
- `--size` flag uses `applySizeOverride(cfg, size)` to set the provider-specific size field (Hetznerâ†’ServerType, DigitalOceanâ†’Size, Lightsailâ†’Bundle) before provider construction; Daytona is a no-op.
- `runSandboxStart` is the public entry point that unwraps `sandboxStartDeps` and delegates to `runSandboxStartWithDeps`; tests can use either path.
- Delete target resolution uses ALL sandboxes (not just running) for `--all`, `--pattern`, and auto-select, unlike stop which filters to running only. Confirmation prompt (`Delete all sandboxes? [y/N]`) is only required for `--all` without `--yes`.
- SSH command uses `parseSSHArgs` for `[NAME] [-- command args...]` parsing since `DisableFlagParsing` prevents Cobra from separating positional args; auto-resolve uses `sandbox.ResolveDefault(nil)` with no filter (any status).
- SSH/exec commands should use injectable `sandboxSSHLoadInstance` and `sandboxSSHResolveProvider` package vars for test isolation; `testMode` flag prevents actual process execution in tests.
- Reuse `sortTargetsByName`, `joinNames`, and `filterRunning` helpers across stop/delete/ssh commands; they live in `cmd/sandbox_stop.go`.
- Status command uses `sandboxStatusLoadInstance`/`sandboxStatusResolveProvider`/`sandboxStatusNow` injectable vars for test isolation; `renderSandboxDetail` reuses `formatAge`/`formatCost` from `sandbox_list.go`.
- When a command changes from `noArgsValidation()` to `cobra.MaximumNArgs(N)`, update `cmd/args_test.go` to remove it from the no-args table.
- `sandbox.Migrate(projectDir, out)` now accepts `io.Writer`; pass `nil` from auto-migrate (no output), pass a `*bytes.Buffer` from the migrate command to detect "no actions" (empty buffer â†’ "Nothing to migrate").
