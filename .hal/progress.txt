# Ralph Progress Log

## Codebase Patterns

- Factory `ArtifactReference` keeps `name` and `type` as required stable fields while richer artifact metadata is additive and optional; keep legacy `path` for current CLI compatibility and use `sourcePath`/`storedPath` for durable collection metadata.
- Factory artifact payloads belong under the global factory store at `artifacts/<run-id>/`; use `Store.SaveArtifactFile` and `Store.ResolveArtifactPath` so writes stay scoped to the store and never create project `.hal` artifact state.
- Local factory artifact collection keeps legacy display `path` values for CLI compatibility while copying payloads into the store with stable `sourcePath` and `storedPath` metadata.
