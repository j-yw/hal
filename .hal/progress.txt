# Ralph Progress Log

## Codebase Patterns

- Factory run/event schemas live in `internal/factory`; exported JSON DTO fields should use explicit camelCase tags and be locked with marshal/unmarshal key tests before storage or CLI layers consume them.
- Factory store paths live in `internal/factory`; the default root is `sandbox.GlobalDir()/factory` with `runs/` and `timelines/` directories created at `0700`, and read-only list operations treat missing directories as empty state.
- Factory run records are persisted through `Store.SaveRun`/`Store.LoadRun` as `runs/<runID>.json`; writes use `0600` temp files plus rename, and missing loads must wrap `fs.ErrNotExist`.
- Use `Store.ListRuns` for full factory run records ordered newest-first by latest created/updated timestamp with run ID as the stable tie-breaker; keep `ListRunIDs` for lexicographic ID-only listings.
- Factory timelines are persisted through `Store.AppendEvent`/`Store.LoadEvents` as `timelines/<runID>.json`; append rewrites the JSON array through the same `0600` temp-file-plus-rename path and `LoadEvents` returns empty state for missing timelines.
- Factory read/list paths should only consider committed `*.json` files via `isCommittedStoreFile`; leftover `*.tmp` and `*.bak` artifacts must stay invisible to run and timeline reads.
