## Codebase Patterns
- Blank imports for database drivers go in `internal/cloud/deploy/drivers.go` â€” centralized driver registration
- `store_factory.go` uses `sql.Open("libsql", ...)` and `sql.Open("pgx", ...)` which depend on driver init() registration
- The deploy package uses `LoadConfig(getenv)` pattern for testability â€” pass `os.Getenv` in production, custom func in tests
- Use `git -c commit.gpgsign=false commit` when GPG signing may not be available
- `DefaultStoreFactory` in `runtime.go` is the composition root â€” matches `func() (cloud.Store, error)` signature used by all cloud command factory vars
- Cloud command store factory vars (e.g., `cloudSubmitStoreFactory`) use if-nil guard in `init()` to allow test overrides
- `godotenv.Load()` in `runCloudEnv` loads `.env` before config validation â€” ignore `os.ErrNotExist`, warn on other errors
- Use `newStoreFactory(func() Config { return LoadConfig(customGetenv) })` to create testable factory instances with isolated `sync.Once` â€” avoids package-level state in tests
